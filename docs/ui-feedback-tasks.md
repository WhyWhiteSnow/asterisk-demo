# План работ по замечаниям к интерфейсу

Документ описывает подзадачи по результатам ревью интерфейса управления ВАТС (Asterisk BATC).

**Стек:** Vue 3 (`ceph-asterisk-front/frontend/`), FastAPI (`ceph-asterisk/`).

**Ключевые файлы:**
- Меню: `frontend/src/components/NavBar.vue`
- Список ВАТС: `frontend/src/views/VatsView.vue`, `frontend/src/components/tables/VatsTable.vue`
- Создание ВАТС: `frontend/src/components/modals/CreateVatsModal.vue`
- Редактирование ВАТС: `frontend/src/components/modals/VatsDetailsModal.vue`
- Внутренние номера: `frontend/src/components/tables/InternalNumbersTable.vue`
- Backend создание: `ceph-asterisk/app/routes/instances/instancesCRUD.py`

## Эпик 3. Создание ВАТС (мастер)

> **Файл:** `frontend/src/components/modals/CreateVatsModal.vue`  
> **Backend:** `ceph-asterisk/app/routes/instances/instancesCRUD.py`

### 3.1. Шаг 2 — только SIP-порт и тип протокола

**Сейчас на шаге 2:** SIP-порт, HTTP-порт, AMI-порт, RTP (начало/конец), тип транспорта.

**Нужно:** оставить только **SIP-порт** и **тип транспорта** (UDP/TCP/TLS).

| ID | Подзадача | Слой | Действие |
|----|-----------|------|----------|
| 3.1.1 | Упростить UI шага 2 | Frontend | Убрать поля HTTP, AMI, RTP из формы |
| 3.1.2 | Автоподбор скрытых портов | Backend | HTTP (8088), AMI (5038), RTP (10000–20000) — автоматически или следующие свободные |
| 3.1.3 | Обновить схему API | Backend | `AsteriskInstanceCreate` — optional для портов, кроме SIP |
| 3.1.4 | Обновить валидацию | Frontend + Backend | UI проверяет только SIP; полная проверка всех портов — на backend |

**Почему:** пользователю не нужно знать AMI/RTP при создании; меньше ошибок и быстрее создание.

### 3.2. Ошибки на русском и привязка к полям

| ID | Подзадача | Проблема | Решение |
|----|-----------|----------|---------|
| 3.2.1 | Русификация ошибок API | Backend отвечает на английском (`Instance name already exists`, `Ports already in use`) | Маппинг `detail` → русский на фронте или русские сообщения в FastAPI |
| 3.2.2 | Field-level ошибки | Общая ошибка порта без указания поля | Ошибка возле конкретного поля; в тексте — название поля |
| 3.2.3 | Ошибки автоподбора портов | После упрощения шага 2 пользователь не видит HTTP/AMI/RTP | Backend возвращает понятное сообщение: «Не удалось подобрать свободный HTTP-порт» |

### 3.3. Подсказка свободных портов / автоподстановка

**Частично реализовано:** `recommendedSipPort` в `CreateVatsModal.vue`.

| ID | Подзадача | Действие |
|----|-----------|----------|
| 3.3.1 | Дефолт в поле SIP-порта | При открытии модалки подставлять `recommendedSipPort`, а не 5060 если он занят |
| 3.3.2 | Список занятых портов | Показать «Заняты: 5060, 5061, 5062» или API `GET /instances/used-ports` |
| 3.3.3 | Кнопка «Использовать рекомендуемый» | Быстро подставить свободный порт в поле |

**Почему:** порт 5060 часто занят — пользователь получает 400/500 без понятной причины.

### 3.4. Чекбокс «Создать тестовых пользователей (101, 102)»

**Сейчас:** hardcoded 101/102 в `instance_pjsip_seed.py`; checkbox → `POST /instances/?create_test_users=true`.

| ID | Подзадача | Действие |
|----|-----------|----------|
| 3.4.1 | E2E-проверка | После создания с галочкой — GET users → должны быть 101 и 102 |
| 3.4.2 | Согласованность UI и seed | Если в шаге 1 появятся другие номера — передавать их в seed, не hardcode |
| 3.4.3 | Обратная связь | Toast: «Созданы пользователи 101, 102» |

**Почему:** нужно подтвердить, что галочка реально создаёт указанные номера.

### 3.5. Ошибка 500 при создании тестовой ВАТС (критично)

> **Приоритет:** P0

Backend: любое необработанное исключение → HTTP 500 с `Failed to create instance: ...`

**Возможные причины:**
1. Docker/compose недоступен
2. Права на файловую систему (`chown`, `chmod`)
3. Ошибка seed PJSIP/voicemail
4. Конфликт портов
5. Ошибки БД (MySQL/CDR)

| ID | Подзадача | Действие |
|----|-----------|----------|
| 3.5.1 | Воспроизвести 500 | Запустить создание в dev, смотреть traceback в логах backend |
| 3.5.2 | Разделить типы ошибок | Конфиг / Docker / БД — разные HTTP-коды и русские сообщения |
| 3.5.3 | Показ detail на фронте | При 500 показывать `detail` из ответа, не только «Ошибка сервера» |
| 3.5.4 | Pre-flight check | Health-check Docker перед созданием (опционально) |

### 5.5. Форма «прилипает» к другим ВАТС

**Проблема:** если не закончить добавление внутреннего номера и закрыть форму, она открывается при просмотре другой ВАТС. Должна повторно открываться только на той ВАТС, для которой планировалось добавление.

**Текущее поведение:**
- `resetForm()` при `show=false` сбрасывает `showAddNumber`
- `currentTab` **не сбрасывается** — при reopen попадаем на вкладку «Номера»
- нет per-VATC draft — черновик не привязан к `instanceId`
- при `show=true` нет сброса UI-состояния перед загрузкой

| ID | Подзадача | Действие |
|----|-----------|----------|
| 5.5.1 | Сброс при смене ВАТС | `watch(vatsData.id)` → reset UI + load data |
| 5.5.2 | Per-VATC draft | `localStorage`: `extension_draft_{instanceId}` (по аналогии с `CreateVatsModal`) |
| 5.5.3 | Сброс вкладки | `currentTab = 'general'` при открытии другой ВАТС |
| 5.5.4 | Правило UX | Черновик восстанавливается **только** для той же ВАТС; для другой — чистая форма |
